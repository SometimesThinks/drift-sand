# drift-sand 기획서

---

## 1. 개요

| 항목          | 내용                                                                                                                                                          |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **목적**      | 포모도로·집중 작업에 맞는 모래시계형 타이머 제공. 숫자만 보는 타이머가 아니라, 남은 시간이 시각적으로 흐르는 것을 보여주어 미니멀하고 직관적인 UX를 지향한다. |
| **타깃**      | 집중이 필요한 개인 사용자(공부, 업무, 휴식 구간 측정 등). MVP는 1인 사용 전제.                                                                                |
| **핵심 가치** | 남은 시간을 한눈에 보는 모래시계 타이머.                                                                                                                      |
| **브랜딩 톤** | 미니멀. 불필요한 텍스트·장식 최소화, 단일 목적(시간 흐름) 강조.                                                                                               |

| ---- | ---------------------------- | -------------- |

---

## 3. v1 범위

**포함 기능**

- 시간 설정: **시/분/초 휠 피커**. **시간 범위 0–24시간은 의도된 범위** (장시간 집중·휴식 시나리오 포함, 최대 24:00:00).
- 시작 / 일시정지 / 리셋.
- 모래시계 시각화: 남은 시간에 비례해 모래가 아래로 흐르는 표현.
- 종료 시 알림: 휴대폰 소리, 휴대폰 진동. (iOS는 알림·진동 정책·권한 제약 확인 후 반영.)
- **백그라운드 동작**: 앱을 내리거나 다른 앱으로 전환해도 타이머는 계속 동작. 포그라운드 복귀 시 진행 상황 유지.

**제외 기능 (향후 검토)**

- 타이머 프리셋 저장·불러오기
- 공유·소셜 기능
- 사용 통계·히스토리

## 4. 수익화·광고

| 항목             | 내용                                                                                                           |
| ---------------- | -------------------------------------------------------------------------------------------------------------- |
| **광고 정책**    | 전면(인터스티셜) 광고 없음.**배너 광고만** 사용.                                                               |
| **배너 위치**    | 하단 고정.                                                                                                     |

---

## 5. 구현 방식

- **선택**: Flutter
- **이유**: 실제 물리감 있는 모래시계 애니메이션을 안정적인 퍼포먼스로 구현하기 유리.

(참고: 웹뷰 래핑·React Native 등 다른 선택지는 초기 검토 후 Flutter로 확정.)

---

## 6. 참고·비고

- **오프라인**: v1 필수 아님. 필요 시 이후 대응 검토.
- **지원 환경**: 1차 Android(구글 플레이) → 2차 Windows(MS Store) → 3차 iOS(앱 스토어).
- **권한**: 알림(소리), 진동 사용. 플랫폼별 권한 요청·설명 문구는 스토어 정책에 맞게 구성.
- **광고·개인정보**: 배너 광고 사용. 스토어 등록 시 **개인정보처리방침** 및 **광고 포함·관련 데이터 처리 안내** 반영 필요.

## 7. v1 핵심 플로우
